# Code Review (Round 2) - URL Shortener Web

Reviewed: 2026-03-26

Second-pass review after the first round of 16 fixes. Only new/remaining issues are listed.

---

## Critical

### 1. `prisma.config.ts` imports removed `dotenv` package

**File:** `prisma.config.ts:2-3`

The file has `import "dotenv/config";` but `dotenv` was removed from `package.json` in the first review round. Any Prisma CLI command (`prisma migrate`, `prisma db push`, `prisma generate`) will crash with `Cannot find module 'dotenv/config'`.

**Fix:** Remove the dotenv import. Prisma's `env()` helper already handles env loading natively.

```diff
- // This file was generated by Prisma, and assumes you have installed the following:
- // npm install --save-dev prisma dotenv
- import "dotenv/config";
  import { defineConfig, env } from "prisma/config";
```

---

### 2. `.env.example` DATABASE_URL uses SQL Server format for a MariaDB project

**File:** `.env.example:1`

The example shows `sqlserver://sa:YourStrong@Password123@localhost:1433;...` but the project uses MySQL/MariaDB (`provider = "mysql"` in schema, `@prisma/adapter-mariadb` in code). Anyone cloning and copying `.env.example` will get a connection error.

**Fix:**
```diff
- DATABASE_URL="sqlserver://sa:YourStrong@Password123@localhost:1433;database=urlshortener;encrypt=true;trustServerCertificate=true"
+ DATABASE_URL="mysql://root:password@localhost:3306/urlshortener"
```

**File:** `src/app/api/auth/[...nextauth]/route.ts:1-2`

  export const { GET, POST } = handlers;

### 4. Register page `res.json()` can crash on empty response

**File:** `src/app/(auth)/register/page.tsx:48`

In the error branch of `handleSubmit`, `await res.json()` is called without `.catch()`. If the server returns an empty body (e.g., 502 from reverse proxy), this throws an unhandled exception - same bug already fixed in `CreateUrlForm.tsx` and `EditUrlModal.tsx` but missed here.

**Fix:**
```diff
- const json = await res.json();
+ const json = await res.json().catch(() => ({}));
```

### 5. Stats route loads ALL clicks into memory

**File:** `src/app/api/urls/[id]/stats/route.ts:16-30`

The route calls `prisma.click.findMany()` to fetch every click from the last 30 days, then groups them in JavaScript. For high-traffic URLs with millions of clicks, this will OOM. Database-side aggregation returns at most 30 rows regardless of volume.

**Fix:** Use raw SQL with database-side aggregation (add a comment referencing the `@@map` so future renames aren't missed):

```ts
// Table name from Click model @@map("shorturl-click") in prisma/schema.prisma
const rows = await prisma.$queryRaw<{ date: string; count: bigint }[]>`
  SELECT DATE(clicked_at) AS date, COUNT(*) AS count
  FROM \`shorturl-click\`
  WHERE url_id = ${id} AND clicked_at >= ${thirtyDaysAgo}
  GROUP BY DATE(clicked_at)
  ORDER BY date ASC
`;
const clicks = rows.map((r) => ({ date: r.date, count: Number(r.count) }));
```

## Low

### 6. Weak password validation

**File:** `src/lib/validations.ts:23`

`registerSchema` only requires `min(8)` with no character diversity. Passwords like `aaaaaaaa` pass validation.

**Fix:** Add complexity rules:
```ts
password: z.string()
  .min(8, "Password must be at least 8 characters")
  .regex(/[A-Z]/, "Must contain at least one uppercase letter")
  .regex(/[a-z]/, "Must contain at least one lowercase letter")
  .regex(/[0-9]/, "Must contain at least one number"),
```

---

### 7. `datasource db` missing `url` in schema.prisma

**File:** `prisma/schema.prisma:6-9`

The datasource block only has `provider = "mysql"` with no `url` field. Some Prisma tooling and IDE extensions expect it.

**Fix:**
```diff
  datasource db {
    provider = "mysql"
+   url      = env("DATABASE_URL")
  }
```
